# Security

Report vulnerabilities privately to the repository maintainers; do not include secrets or private collection data in a public issue. Collectr/Dex cookies, tokens, browser profiles, exports, grading certification numbers, and collection state are sensitive.

Authentication remains in user-controlled local sessions. Debug fixtures must be fictional or minimized and sanitized. CardRelay does not support CAPTCHA, access-control, rate-limit, anti-bot, or stealth-automation bypasses. Browser profiles carry account-access risk: protect the local OS account and clear profiles when no longer needed.
